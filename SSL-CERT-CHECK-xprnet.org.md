## Root Cause

GitHub Pages has **not yet provisioned an SSL certificate** for the custom domain `xprnet.org`.

When a user visits `https://xprnet.org`, GitHub Pages serves its default `*.github.io` certificate instead of a certificate specifically for `xprnet.org`.

---

## Why This Happens

GitHub Pages automatically requests Let's Encrypt certificates for custom domains, but this process can:
- Take up to **24 hours** after DNS is configured
- Fail if DNS is not properly set up
- Require manual re-trigger if the domain was recently added/changed

---

## Solutions (in order)

### Option 1: Wait (Recommended first)
If xprnet.org DNS was recently pointed to GitHub Pages, **wait up to 24 hours** for GitHub to automatically provision the SSL certificate.

### Option 2: Re-trigger SSL provisioning in GitHub
1. Go to: `https://github.com/Melxisedek75/gcsc-smart-contractor/settings/pages`
2. In the **Custom domain** section:
   - Remove `xprnet.org`
   - Save
   - Wait 1 minute
   - Re-add `xprnet.org`
   - Save again
3. GitHub will re-request the Let's Encrypt certificate
4. Wait 5-15 minutes
5. Check `https://xprnet.org` — certificate should now show `CN=xprnet.org`

### Option 3: Check GitHub Pages is enabled
1. Go to repo Settings → Pages
2. Ensure **Source** is set to `Deploy from a branch`
3. Ensure branch is `main` and folder is `/ (root)` or `/docs`
4. Ensure **Enforce HTTPS** is checked (after certificate is ready)
